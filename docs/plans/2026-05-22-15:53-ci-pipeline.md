# CI Pipeline — GitHub Actions

## Context

The project has no `.github/workflows/` directory. Lint, typecheck, and all three test layers (unit, e2e) must be run manually. Broken changes can land on `main` undetected. This plan adds a single GitHub Actions workflow to close that gap.

## Decisions

- **Triggers:** `push` to `main` + `pull_request` targeting `main`
- **Node version:** 20 LTS (pinned via `actions/setup-node@v4`)
- **Database:** SQLite — `DATABASE_URL=file:./ci.db` in the job env; uses the default `prisma/schema.prisma`
- **LLM review tests:** included — `llm-review.spec.ts` is fully mocked via `page.route()`, no `OPENROUTER_API_KEY` needed
- **E2e servers:** Playwright's `webServer` config already starts `npm run dev` and `npm run dev:ws` automatically; no manual server step needed

## File to create

`.github/workflows/ci.yml` (new file)

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  ci:
    runs-on: ubuntu-latest

    env:
      DATABASE_URL: file:./ci.db
      AUTH_SECRET: ci-secret-not-for-production

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Generate Prisma client
        run: npx prisma generate

      - name: Run database migrations
        run: npx prisma migrate deploy

      - name: Lint
        run: npm run lint

      - name: Type check
        run: npm run typecheck

      - name: Unit tests
        run: npm run test -- --run

      - name: Install Playwright browsers
        run: npx playwright install --with-deps chromium

      - name: E2e tests
        run: npm run test:e2e
```

## Notes

- `AUTH_SECRET` is a hardcoded dummy value — valid enough for NextAuth to initialise in a test environment, not a real secret
- `npx prisma generate` must run before migrations; `npm ci` does not automatically generate the client
- `cache: 'npm'` speeds up repeated runs by caching the npm cache directory keyed on `package-lock.json`
- Playwright's `reuseExistingServer: true` means local runs stay unaffected; CI always gets fresh servers

## Verification

1. Push a branch, open a PR — the Actions tab should show the workflow running
2. Confirm all steps pass: lint → typecheck → unit tests → e2e tests
3. Introduce a deliberate lint error on a branch and confirm CI fails on the lint step
