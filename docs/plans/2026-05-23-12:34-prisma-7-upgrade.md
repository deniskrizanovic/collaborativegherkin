# Prisma 7 Upgrade Plan

## Context

The project currently runs Prisma 6 (`^6.0.0`). Prisma 7 is a major release with several
breaking changes that require coordinated edits across the schema files, client initialisation,
import paths, and CLI scripts. This plan covers every breaking change that applies to this
codebase and nothing beyond.

**Good news — already satisfied:**
- `"type": "module"` is already in `package.json` (ESM ✓)
- `tsconfig.json` already has `module: esnext`, `moduleResolution: bundler` (TS config ✓)
- Node 22.x in use locally; CI is on Node 20 (needs patch, see step 8)
- No `$use()` middleware to migrate
- No transactions to worry about
- No `directUrl` in schemas

---

## Files to change

| File | Change |
|------|--------|
| `package.json` | Bump `prisma`/`@prisma/client` to `^7`; add adapters; remove `"prisma"` seed block |
| `tsconfig.json` | Bump `target` from `ES2017` → `ES2023` |
| `prisma/schema.prisma` | New generator provider + output; remove `url` from datasource |
| `prisma/postgres/schema.prisma` | Same as above |
| `prisma.config.ts` *(new)* | SQLite config for default CLI use |
| `prisma/postgres/prisma.config.ts` *(new)* | Postgres config for prod CLI use |
| `src/lib/db.ts` | New import path; add driver adapter logic |
| `prisma/seed.ts` | New import path; add driver adapter |
| `src/lib/session.ts` | Update `type { PrismaClient }` import path |
| `src/lib/coaching.ts` | Update `type { PrismaClient }` import path |
| `.github/workflows/ci.yml` | Pin Node to `22` (Prisma 7 requires ≥ 20.19.0; CI is on `'20'`, which may resolve below that) |

## Implementation steps

### 1 — Install packages

```bash
npm install @prisma/client@^7 prisma@^7 \
  @prisma/adapter-better-sqlite3 \
  @prisma/adapter-pg \
  dotenv

# update auth adapter (not yet active in code, but keep it compatible)
npm install @auth/prisma-adapter@latest
```

`@prisma/adapter-better-sqlite3` is needed for dev/CI (SQLite).
`@prisma/adapter-pg` is needed for production (Postgres).
`dotenv` is needed by `prisma.config.ts` at CLI time (not bundled by Next.js).

---

### 2 — Update `tsconfig.json`

Change `target` from `"ES2017"` to `"ES2023"`.

---

### 3 — Update `prisma/schema.prisma`

```prisma
generator client {
  provider = "prisma-client"
  output   = "../src/generated/prisma"
}

datasource db {
  provider = "sqlite"
}
```

- `provider` changes from `prisma-client-js` → `prisma-client`
- `output` is now required; path is relative to the schema file → resolves to `src/generated/prisma`
- `url` is removed from the datasource block (moves to `prisma.config.ts`)

---

### 4 — Update `prisma/postgres/schema.prisma`

```prisma
generator client {
  provider = "prisma-client"
  output   = "../../src/generated/prisma"
}

datasource db {
  provider = "postgresql"
}
```

Same rationale as above. Output path from `prisma/postgres/` needs an extra `../` to reach project root.

---

### 5 — Create `prisma.config.ts` (project root, SQLite default)

```ts
import "dotenv/config";
import { defineConfig, env } from "prisma/config";

export default defineConfig({
  schema: "prisma/schema.prisma",
  datasource: { url: env("DATABASE_URL") },
});
```

This is the config used by all default CLI invocations (`prisma migrate dev`, `prisma generate`, etc.).

---

### 6 — Create `prisma/postgres/prisma.config.ts`

```ts
import "dotenv/config";
import { defineConfig, env } from "prisma/config";

export default defineConfig({
  schema: "prisma/postgres/schema.prisma",
  datasource: { url: env("DATABASE_URL") },

```

Used by Postgres-specific CLI commands via `--config prisma/postgres/prisma.config.ts`.

### 7 — Update `package.json`

**Remove** the `"prisma"` block (no longer supported in v7):
```json
// delete this:
"prisma": {
  "seed": "npx tsx prisma/seed.ts"
}
```

**Update Postgres scripts** to pass `--config`:
```json
"db:generate:postgres": "prisma generate --config prisma/postgres/prisma.config.ts",
"db:migrate:dev:postgres": "prisma migrate dev --config prisma/postgres/prisma.config.ts",
"db:migrate:prod": "prisma migrate deploy --config prisma/postgres/prisma.config.ts"
```

**Add explicit generate step** where needed (v7 no longer auto-generates after `migrate dev`):
```json
"db:migrate:dev:postgres": "prisma migrate dev --config prisma/postgres/prisma.config.ts && prisma generate --config prisma/postgres/prisma.config.ts"
```

The `"seed"` npm script is unaffected.

### 8 — Update `.github/workflows/ci.yml`

Change Node version from `'20'` to `'22'`:

```yaml
- uses: actions/setup-node@v4

    node-version: '22'
```

Also add explicit `prisma generate` after migrations (v7 doesn't auto-generate):


The CI already has a separate generate step before migrations, so the order is already correct —
just verify nothing silently drops it.

### 9 — Update `src/lib/db.ts`

```ts
import { PrismaClient } from "../generated/prisma/client";
import { PrismaBetterSqlite3 } from "@prisma/adapter-better-sqlite3";
import { PrismaPg } from "@prisma/adapter-pg";

const globalForPrisma = globalThis as unknown as { prisma: PrismaClient };

function createPrismaClient(): PrismaClient {
  const url = process.env.DATABASE_URL ?? "";
  const adapter = url.startsWith("file:")
    ? new PrismaBetterSqlite3({ url })
    : new PrismaPg({ connectionString: url });

  return new PrismaClient({
    adapter,
    log: process.env.NODE_ENV === "development" ? ["error", "warn"] : ["error"],
  });
}

export const db = globalForPrisma.prisma ?? createPrismaClient();

if (process.env.NODE_ENV !== "production") globalForPrisma.prisma = db;
```

The `DATABASE_URL` prefix (`file:`) distinguishes SQLite from Postgres at runtime.

### 10 — Update `prisma/seed.ts`

```ts
import { PrismaClient } from "../src/generated/prisma/client";
import { PrismaBetterSqlite3 } from "@prisma/adapter-better-sqlite3";
import { DEFAULT_PROMPT, DEFAULT_MODEL } from "../src/lib/llm-constants";

const url = process.env.DATABASE_URL ?? "file:./prisma/dev.db";
const adapter = new PrismaBetterSqlite3({ url });
const db = new PrismaClient({ adapter });
```

Seed always runs against SQLite (dev/CI only), so the adapter is hardcoded to `PrismaBetterSqlite3`.
Rest of the file is unchanged.

---

### 11 — Update `src/lib/session.ts` and `src/lib/coaching.ts`

Both files import only the `PrismaClient` type. Change the import path:

```ts
// before
import type { PrismaClient } from "@prisma/client";

// after
import type { PrismaClient } from "../generated/prisma/client";
```

No logic changes; these are type-only imports used for dependency-injection typing.

---

### 12 — Run Prisma generate

After all schema and config changes are in place:

```bash
npx prisma generate
```

This writes the generated client to `src/generated/prisma/`. Commit the generated directory
(or `.gitignore` it and generate in CI — current CI already has a generate step, so either works).
